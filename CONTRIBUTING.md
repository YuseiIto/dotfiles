# Contributing

First off — thanks for even thinking about it. 🙏

This is primarily a personal dotfiles repository, so there's no roadmap or feature requests backlog to work from. That said, contributions are genuinely welcome if you spot something worth fixing.

## What's worth a PR

- **Bug fixes** — Something broken in a cookbook, a typo, or a broken shell script? Go for it.
- **Platform compatibility** — Found an issue on Debian/Ubuntu or macOS? Fixes are very appreciated.
- **Documentation** — If something confused you, it'll probably confuse the next person too.

## What probably isn't

- New tools or packages you personally use — those are better suited for your own fork.
- Opinionated config changes (keybindings, colorschemes, etc.) — these are pretty deliberate choices.

## How to contribute

Issues are welcome too — even if you're not up for a PR, pointing out bugs or quirks is genuinely helpful.

For code contributions:

1. Fork the repo and create a branch.
2. Make your change. Keep it focused — one fix per PR.
3. Check the code style: `make lint`, then `make format` to auto-fix.
4. Open a PR with a short description of what and why.

No need to open an issue first for small fixes — just send the PR.

## A note on scope

Since this is a personal environment, I might close PRs that don't fit my workflow — nothing personal. But if you're on the fence about whether something is worth contributing, feel free to open an issue and ask first.
