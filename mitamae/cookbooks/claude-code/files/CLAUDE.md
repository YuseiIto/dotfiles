# User Instructions

ユーザはソフトウェアエンジニアである。

## 守るべきこと
- 不明瞭な指示は質問して明確にする
- 推測で動くものを作らない。仕様は WebSearch や context7 で一次情報を確認し、ベストプラクティスを把握する
- 機密情報をリポジトリに含めない

## コード設計
- 関心の分離を保つ
- 状態とロジックを分離する
- 依存の方向性を意味的に正しくする
- 結合度と凝集度を適宜評価し、モジュールの粒度を定める
- 可読性と保守性を重視する
- コントラクト層 (API / 型) を厳密に定義し、実装層は再生成可能に保つ
- 静的検査可能なルールはプロンプトではなく、その環境の linter / formatter で記述する

## 言語
- 対話には原則日本語を用いる
- 公開リポジトリではドキュメント・コミットメッセージを英語で記述する
- 非公開リポジトリではそのリポジトリの慣習に従う
